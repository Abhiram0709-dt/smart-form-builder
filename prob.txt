PROBLEMS FACED & SOLUTIONS

--------------------------------------------------

1. Node.js Version Compatibility Issue

Problem:
While running the React frontend using Vite, the application failed with an error indicating that the current Node.js version (v20.18.0) was not supported.

Error Message:
"Vite requires Node.js version 20.19+ or 22.12+"

Root Cause:
The installed Node.js version was slightly outdated and did not meet Vite’s minimum requirement.

Solution:
Upgraded Node.js to the latest LTS version (v22.x.x) from the official Node.js website.

Verification:
Checked using:
node -v

--------------------------------------------------

2. Missing Native Binding (Rolldown Error)

Problem:
After upgrading Node, the application still failed with an error related to missing native bindings.

Error Message:
"Cannot find native binding… npm has a bug related to optional dependencies"

Root Cause:
Corrupted or incomplete installation of dependencies in node_modules due to npm optional dependency issues.

Solution:
Performed a clean reinstall of dependencies:

Steps:
1. Deleted node_modules directory
2. Deleted package-lock.json
3. Reinstalled dependencies using:
   npm install

After reinstall, the issue was resolved.

--------------------------------------------------

3. Environment Setup Decision

Challenge:
Choosing between Docker-based setup vs local Node.js installation.

Decision:
Chose local Node.js setup to reduce overhead and speed up development.

Reason:
Since this is a 3–4 hour prototype task, minimizing setup complexity was more important than containerization.

--------------------------------------------------

4. Learning / Key Takeaways

- Always verify tool version compatibility before debugging deeper issues
- Native binding errors are often caused by corrupted node_modules
- Cleaning and reinstalling dependencies is a reliable first step
- For rapid prototyping, prefer simple local setups over complex environments like Docker
